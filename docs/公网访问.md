# SPCX 套利监控 · 让别人访问

当前完整功能（7 所行情 + 价差矩阵 + 42 组历史 K 线）依赖本机运行的 **`python3 server.py`**。  
`127.0.0.1:8080` 只有你自己能打开，外网/别人需要下面任一方式。

---

## 方式一：临时公网链接（最快，适合发给朋友试用）

适合：不想买服务器，Mac 开着、服务在跑。

**终端 A**（保持运行）：

```bash
cd spcx-arbitrage-monitor
./start.sh
```

**终端 B**：

```bash
cd spcx-arbitrage-monitor
chmod +x share.sh
./share.sh
```

或手动：

```bash
brew install cloudflare/cloudflare/cloudflared
cloudflared tunnel --url http://127.0.0.1:8080
```

终端会打印类似 **`https://随机名.trycloudflare.com`**，把这个链接发给别人即可（含监控页与 `/history.html`）。

注意：

- 关掉终端 B 或 Mac 休眠后，链接会失效
- 每次重启隧道，域名可能变化
- **无登录密码**，知道链接的人都能看，勿公开敏感环境

### 隧道报错 `TLS handshake … EOF` 且 IP 为 `198.18.0.x`

说明 **Clash / Surge 等 TUN 代理** 劫持了 `cloudflared` 连 Cloudflare 的流量（`198.18.0.0/15` 是 Fake IP 段）。

**按顺序试：**

1. 在 Clash Verge / Surge 里 **关闭 TUN / 增强模式**（或暂时退出代理），再执行 `./share.sh`
2. 或在规则里加 **直连**：进程名 `cloudflared`；域名 `*.cloudflare.com`、`*.trycloudflare.com` → `DIRECT`
3. 本仓库 `share.sh` 已改用独立配置 `share-tunnel-min.yml`（`quic`），勿只靠 `~/.cloudflared/config.yml` 里的 `protocol: http2`

连上后日志应出现 `Registered tunnel connection`，公网链接才可访问。

## 方式二：同一 WiFi / 局域网

适合：办公室、家里同一网络下的同事。

```bash
HOST=0.0.0.0 PORT=8080 python3 server.py
```

查本机 IP（Mac）：

```bash
ipconfig getifaddr en0
```

别人浏览器访问：`http://你的IP:8080`（例如 `http://192.168.1.23:8080`）。

注意：路由器需允许设备互访；**不能**让外网用户访问（除非再做端口映射，不推荐家用路由）。

---

## 方式三：云服务器长期在线（推荐正式对外）

适合：24h 给别人用、要固定域名。

1. 买一台 VPS（阿里云 / 腾讯云 / AWS 等），系统 Ubuntu
2. 把项目拷到服务器，安装 Python 3.9+
3. 安装依赖并启动：

```bash
cd spcx-arbitrage-monitor
pip3 install -r requirements.txt
HOST=0.0.0.0 PORT=8080 python3 server.py
```

4. 用 **Nginx + HTTPS** 反代到 8080，绑定域名（Let’s Encrypt）
5. 用 **systemd** 让 `server.py` 开机自启、崩溃重启
6. 历史数据在服务器 `data/spread_ticks.ndjson`，需保证磁盘空间

安全建议：加 Nginx 基础认证或 IP 白名单，避免行情页对全网公开。

---

## 方式四：Cloudflare Workers（仅 API 轻量）

项目里有 `npm run deploy`（Workers + `web/` 静态页），但：

- Worker **内存不持久**，价差历史 K 线落盘能力弱于本机 `server.py`
- 完整 42 组历史 + 长期积累，仍建议 **VPS 跑 server.py** 或 **隧道 + 常开电脑**

若只要监控页、不要长期历史，可尝试 Workers；否则用方式一或三。

---

## 常见问题

| 问题 | 处理 |
|------|------|
| 别人打不开 | 确认 `./start.sh` 在跑；隧道是否还在；防火墙是否拦 8080 |
| 历史 K 线很短 | 服务器需持续运行，且有人访问过监控页才会写入 tick |
| 想固定域名 | 用 VPS + 域名，或 Cloudflare 命名隧道（需 Cloudflare 账号） |
